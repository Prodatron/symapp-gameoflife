1000-Introduction


INTRODUCTION TO CONWAY'S GAME OF LIFE

The Game of Life, also known simply as Life, is a cellular automaton devised by the British mathematician John Horton Conway in 1970. It is a zero-player game, meaning that its evolution is determined by its initial state, requiring no further input. One interacts with the Game of Life by creating an initial configuration and observing how it evolves.

Beside the basic rules and the cell field editor this implementation for SymbOS provides several additional features. A large pattern library is available for the editor as well as a set of complete predefined worlds.

Unlike most other Game of Life implementations it is possible to define different rules for the death and the birth of cells depending on the amount of neighbour cells.

SCREEN LAYOUT

Cell field

The cell field is the large area at the upper left side of the window. It shows the current cell population, black dots are living cells. If Game Of Life is in pause mode you can edit the cell field by clicking into it. Depending on the edit mode, you will either add or remove a single cell or place a predefined pattern at the mouse position.

Dead/alive rule settings

The rule settings are placed at the bottom of the window.

The checkboxes in the first line define, how many neighbours are necessary for a cell to stay alive. In the classic Game of Life 2 or 3 neighbours are required for a cell to stay alive. If there are less or more a cell will disappear. So only checkboxes 2 and 3 are active.

The checkboxes in the second line define, how many neighbours are necessary for the birth of a new cell. The classic rules specify, that only when there are 3 neighbours a new cell will appear.

By checking/unchecking the checkboxes you can alter the rules. This can end up in very stupid but also very interesting results.

The "copy world" is a famous variation, just check all odd-numbered checkboxes (1,3,5,7) in both lines. See also the predefined copy world in the PRESETS menu.

By checking 1,2,3,4 for the first and 2 for the second your cells will usually end up in a static labyrinth-like structure.

2200-Pull Down Menus


FILE MENU

Here you can load an existing cell field or saves the current one to a file.


PRESETS MENU

Here you can load complete predefined worlds. Most are examples for different kind of patterns for the classic Game Of Life.

Oscillators are patterns, which may change the shape a lot but after some generations they will return to their starting shape.

Glider guns are producing so called "gliders" which are small moving patterns.

Spaceships are very large patterns, which are still stable like Oscillators but moving as well either diagonal or vertical/horizontal.

1357 copy world is an example for an alternative rule set.
